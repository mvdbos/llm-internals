Lesson 1: Tensors & Layers

The building blocks of an LLM — what tensors and layers actually are.

What's a tensor?

A tensor is a grid of numbers. You already know them by other names:

From a single number to a cube of numbers — they're all tensors
DimensionNameWhat it looks likeExample
0D Scalar A single number 42
1D Vector A row of numbers [0.2, 0.8, -0.5]
2D Matrix A table of numbers A spreadsheet grid
3D+ Tensor A cube (or more) of numbers A stack of spreadsheets

In practice, ML people call everything a tensor — scalars, vectors, matrices, all of it. The word just means "numbers flowing through the model."

Think of it like a factory conveyor belt. Each item on the belt is a tray with compartments holding numbers — that tray is a tensor. The model moves trays from station to station, and each station transforms the numbers in some way.

What's a layer?

A layer is one of those stations. It takes a tensor in, does math to it, and spits a (usually different) tensor out.

Input tensor  →  [ Layer: math operations ]  →  Output tensor

What math? Mostly two things:

  1. Multiply the input numbers by a set of stored numbers called weights
  2. Add some more stored numbers called biases

The weights are the "knowledge" of the layer — they're what the model learned during training. When you download an LLM, you're downloading billions of these weight numbers.

Key idea

Weights are the learned numbers that a layer uses when it transforms its input. A model stores many of them across its layers. Quantization changes how selected stored weights are represented; it does not change the basic job of the layers.

From input to output

An LLM applies a sequence of layers to numeric input representations. Each layer uses stored weights to transform those numbers, and later layers use the resulting values. For this course, the important point is simple: weights are stored numbers, and changing their representation can change storage and approximation.

Why this matters for quantization

Quantization represents selected stored weights with a more compact numeric encoding. Fewer code bits can reduce weight storage, but the reconstructed values are approximations. This is a simplified weight-code illustration, not a model-file or runtime-memory forecast: real artifacts also contain reconstruction metadata and other components.
