Lesson 3: Will It Fit? Model Memory and Context

A smaller model file helps, but file size is not the amount of memory a running model needs. To decide whether a candidate will fit, separate what is loaded once from what grows while you use it.

1. Four memory buckets

  1. Artifact/file size: bytes in the download.
  2. Loaded weights: model tensors allocated by a named loading policy.
  3. Retained runtime state: state kept while processing a workload.
  4. Temporary workspace and runtime overhead: working buffers, allocator overhead, and other transient allocations.
peak runtime memory
≈ loaded weights
+ retained runtime state
+ temporary workspace and overhead

This is an accounting estimate, not a universal allocator formula. Measure peak memory under the engine and workload you actually plan to use.

2. Context can change the peak

Fictional example: the same 4 GB artifact can have different runtime peaks
RunWhat changesExpected effect
Short-context runLess workload-dependent retained stateLower peak memory
Long-context runMore workload-dependent retained stateHigher peak memory

The file and loaded parameters did not grow. The allocation associated with the workload changed.

In conventional cached attention, retained state is often a KV cache. Other architectures retain different state, so do not apply a KV-cache formula everywhere.

3. Record a fit check

  • Artifact size.
  • Loaded or idle memory under a named inference engine and loading policy.
  • Prompt length, generated length, and batch.
  • Peak runtime memory during a representative run.
  • Remaining headroom.

Headroom is practical insurance: a candidate that barely fits one short prompt may fail when context, outputs, or temporary allocations grow.

Quiz: check the memory boundary

1. Why is artifact size not peak memory?

2. Why can the same artifact have different peaks?

3. What must be recorded before claiming a candidate fits?

Takeaway and next step

Measure a candidate under the workload you need, and keep headroom rather than relying on download size. Next, identify the concrete artifact, quantization recipe, and runtime whose fit you are checking.
