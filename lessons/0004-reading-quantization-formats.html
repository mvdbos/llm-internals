Lesson 4: Reading Quantization Formats

A filename such as Q4_K_M looks like an answer, but it is only a label. Before comparing downloads, identify what category each name belongs to and what information is still missing.

1. One name, one category

Scroll horizontally to inspect every category.

Representative implementation examples. Exact version-pinned details and sources live in the optional format reference.
ExampleCategoryDoes not establish
GGUFArtifact/containerEncoding of every tensor or runtime performance
Q4_K_MModel-file recipeOne universal four-bit tensor representation
MLX affine 8-bit/group 64Quantization configurationGGML Q8_0, retained-state precision, or universal compatibility

Those categories answer different questions. An artifact packages tensors and metadata. A recipe can assign different encodings to different tensor roles. A quantization configuration describes a particular target, mapping, code width, and grouping. None of them names the inference engine that must load and execute the candidate.

2. Inspect fields, not just filenames

Use this fictional configuration record as a reading exercise:

checkpoint: selected-checkpoint-revision
artifact/container: GGUF
quantized target: eligible weights
encoding/recipe and grouping: Q4_K_M; inspect tensor metadata
runtime/backend version: named-runtime revision
intended hardware: declared device

From this record you can describe the candidate. You still cannot conclude that it fits with enough context, uses efficient kernels, is fast enough, or preserves task quality. The memory lesson established what to measure for fit; the choice lesson shows how to validate compatible candidates.

3. Produce a complete candidate description

checkpoint
+ artifact/container
+ quantized target
+ encoding/recipe and grouping
+ runtime/backend version
+ intended hardware

Record the complete description before comparing results. Two labels with a similar number of bits can differ in metadata, tensor coverage, supported kernels, or runtime behavior. A successful load proves only basic loader compatibility—not peak memory, speed, or useful output quality.

4. Keep optional detail in the reference

You do not need to memorize every encoding family to choose a candidate. The quantization formats reference preserves version-pinned catalogues, effective-bits-per-weight examples, and runtime/kernel boundaries for when you need to inspect a specific implementation.

Quiz: classify before comparing

1. What category is GGUF?

2. Why are similarly named eight-bit formats not automatically interchangeable?

3. A smaller artifact loads successfully. What remains unproven?

Takeaway and next step

A label becomes useful only when you can name its category and fill in the missing fields. Now that you can describe compatible candidates precisely, the next lesson shows how much validation is appropriate before choosing.
