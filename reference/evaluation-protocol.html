Controlled evaluation protocol

This optional reference is for a public, expensive, or causal claim about quantization. It is not required for every local-model choice.

Keep statement types distinct

Evidence required for each kind of claim
TypeEvidence
MechanismMathematics or a high-trust technical source
Implementation detailPinned code, documentation, and artifact metadata
ObservationRecorded measurement under a declared protocol
HypothesisAn explicit label plus a discriminating follow-up test

Preregister the comparison

Hold checkpoint, task, runtime settings, prompts/tools, environment, generation settings, and verifier revision fixed. Define expected task × repeat × condition cells, missing-data rules, exclusions, reruns, and outcome metrics before looking at results.

Use matched, repeated conditions

Run each task under each condition. Repeat stochastic settings. Report failures instead of silently dropping them. Pooled averages from unmatched tasks or mixed architectures cannot isolate a quantization effect.

Report quality before efficiency

Choose workload-appropriate quality measures: verified tests for coding, accuracy/F1 for classification, factuality criteria for summarization, a defined rubric or blinded preference for writing, and externally verified completion for agents. Interpret memory and speed alongside those outcomes.
