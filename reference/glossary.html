LLM and quantization glossary

Terms are separated by role: architecture, model artifact, quantized target, inference engine, runtime state, kernels/hardware, and evaluation. Architecture-dependent definitions say which architecture they describe.

ABCDEFGHIKLMOPQRSTVWZ

A

Activation
runtime stateA value produced while the model processes input. Activations are distinct from stored parameters: parameter quantization does not automatically quantize activations.
Affine Quantization
quantizationA linear mapping whose reconstruction grid has both a scale and a shift, commonly written q = round(x / s) + z and x̂ = s(q − z). The integer zero point z, a reconstruction offset/bias, and the scale are related but mathematically distinct quantities.
Architecture
architectureThe model's computational structure: its layer types, dimensions, attention or recurrence pattern, normalization, routing, and other operations. An artifact records tensors and metadata for an architecture; an engine implements those operations.
Artifact model artifact, container
artifactA distributable file or directory that stores model tensors plus enough metadata to reconstruct them. GGUF and an MLX model directory are different artifact families; neither is itself an inference engine.
Attention self-attention when source and target are the same sequence
architectureA mechanism that forms weighted combinations of value vectors using similarities between query and key vectors. Causal self-attention masks future positions. Windowed, grouped-query, multi-query, linear, and hybrid variants do not all have the same compute or retained-state formula.
Attention Head
architectureOne query-head partition and its attention calculation. In standard multi-head attention each query head has corresponding key/value heads; in grouped-query or multi-query attention, multiple query heads share fewer key/value heads. Query-head count, KV-head count, and head dimension are separate architecture fields.

B

Backend
runtimeThe execution path used by an inference engine for a device or framework. A backend determines available kernels, supported tensor encodings, and often performance; support must be checked for a named version.
bf16 bfloat16, Brain Floating Point
numeric formatA 16-bit floating-point format with one sign bit, eight exponent bits, and seven fraction bits. Compared with FP16 it keeps FP32-like exponent range but less significand precision.
Bias
architectureA learned additive term in an operation such as y = Wx + b. Some architectures omit bias terms in some or all projections.
Block Size
quantized targetThe number of values that share encoding metadata in a block format. It is scheme-specific and not interchangeable with an MLX affine group size unless the implementation defines them identically.
Block-wise Quantization
quantizationA family of encodings that partition a tensor into blocks and store shared metadata—such as scales, offsets, or codebook information—alongside low-bit values. Effective storage includes both codes and metadata.

C

Calibration Data
quantizationRepresentative inputs used by some post-training methods to estimate sensitivity, ranges, or reconstruction error. Methods that use no calibration data should not be described as calibration-based.
Causal Mask
architectureA mask that prevents a sequence position from attending to future positions in conventional causal attention. Implementations may represent it explicitly or enforce the same restriction inside a kernel.
Checkpoint
modelA specific saved model state. A fair precision comparison starts from the same checkpoint and changes only the declared conversion or quantization treatment.
Context Window
architecture/runtimeThe sequence length a particular model and engine configuration can process. Advertised maxima, configured maxima, and practical limits can differ; retained-state memory usually grows with sequence length but the formula depends on architecture and cache strategy.

D

Decoding
generation settingThe rule that selects output tokens from logits or probabilities, including greedy choice, sampling, constraints, and stop handling. Decoding settings can change observed behavior without changing model weights.
d_model hidden dimension
architectureThe width of a token representation in many Transformer configurations. Other dimensions—attention head width, intermediate width, recurrent state, or expert width—must be read separately from the model configuration.

E

Effective Bits per Weight effective bpw
measurementTotal model-artifact tensor bytes multiplied by eight and divided by the declared parameter count, under an explicitly stated accounting rule. It can differ from the nominal code width because metadata, mixed tensor encodings, and unquantized tensors add storage.
Embedding
architectureA vector representation associated with a token or other discrete item. Similarity is learned from training; no individual coordinate has a fixed human-readable meaning.
Embedding Layer
architectureA parameterized lookup that maps token IDs to vectors. Whether it is quantized, tied to the output projection, or kept at another precision is an artifact/conversion decision.

F

Feed-Forward Network FFN, MLP block
architectureA per-position nonlinear transformation used in many Transformer layers. Its expansion ratio, activation, gating, expert routing, parameter share, and quantization sensitivity are architecture- and checkpoint-dependent.
Fail-to-Pass F2P
evaluationThe share or count of task-specific tests that failed before a change and pass afterward. The exact test set, denominator, verifier revision, and treatment of missing results must be reported.
FP16 IEEE binary16, half precision
numeric formatA 16-bit floating-point format with one sign bit, five exponent bits, and ten fraction bits. Calling it a “quality baseline” is an experiment choice, not a mathematical guarantee.
FP32 IEEE binary32, single precision
numeric formatA 32-bit floating-point format with one sign bit, eight exponent bits, and 23 explicitly stored fraction bits. Engines may use it selectively for accumulation or numerically sensitive operations.

G

GGUF GGML Universal File
artifactA binary model-artifact format containing metadata and typed tensors from the GGML type system. It belongs to the artifact/container layer; it is not an inference engine, a hardware backend, or a synonym for quantization.
GGUF quantization type and model-file recipe labels
quantized targetLabels such as Q8_0, Q6_K, and IQ4_XS name concrete GGML tensor encodings. Labels such as Q4_K_M can name a llama.cpp model-file recipe that selects a mixture of tensor encodings. Meanings and effective bpw are version- and model-specific.
Group Size
quantized targetThe number of values that share quantization parameters in a group-wise implementation. Always state it with mode and bit width—for example, affine 8-bit/group 64.
Group-wise Quantization
quantizationQuantization that shares parameters across a group of values. “Group” and “block” are implementation terms; use the upstream API's name rather than assuming two schemes are equivalent.

H

Hardware
executionThe CPU, GPU, accelerator, memory subsystem, and device topology on which kernels run. Artifact size alone does not predict speed; hardware and kernel support are part of the comparison.
Hidden State
runtime stateAn intermediate representation passed between operations. In a conventional Transformer it commonly has shape (batch, sequence, hidden_dimension); recurrent and state-space models may retain different state.

I

Importance-Weighted Quantization IQ in GGML labels
quantized targetA family of GGML encodings and quantizer procedures that use importance information and specialized codebooks/metadata. Quality must be evaluated for the exact tensor type, recipe, model, and workload; “IQ” is not a generic guarantee of superiority.
Inference
runtimeExecuting a trained checkpoint to obtain outputs. Inference memory includes loaded parameters and runtime allocations; it is not equal to artifact file size.
Inference Engine runtime
runtimeSoftware that loads an artifact, schedules model operations, manages retained state, and invokes device kernels. Engine name and version are required when claiming format support or speed.

K

KL Divergence Kullback–Leibler divergence
evaluationAn asymmetric measure of difference between probability distributions, commonly KL(p‖q) = Σ pᵢ log(pᵢ/qᵢ). In model comparisons it is an intrinsic distribution diagnostic, not proof of equal downstream behavior.
K-quant
quantized targetA GGML family of super-block tensor encodings such as Q4_K and Q6_K. A model-file recipe with an _M suffix may choose different encodings for different tensors; it is inaccurate to reduce that selection to “attention gets one bit width and FFN another.”
Kernel
executionA low-level implementation of an operation for particular hardware and tensor layouts. A compact encoding can be slower if the engine lacks an efficient kernel for it.
KV Cache
runtime stateIn conventional cached self-attention, retained key and value vectors from processed tokens. It does not contain attention scores and old entries are normally reused rather than rewritten. Memory depends on layers, retained KV heads, head dimension, sequence length, bytes per element, batch, and cache strategy. Recurrent, state-space, linear-attention, and hybrid models require different retained-state descriptions.

L

Layer
architectureA named stage of computation that transforms inputs into outputs and may contain parameters. “Layer” can refer to different granularities, so memory calculations should name the exact tensors counted.
Layer Normalization
architectureA normalization operation over feature dimensions. Modern architectures may use LayerNorm, RMSNorm, or variants and may place them before or after other blocks.
Logits
runtime stateUnnormalized scores produced before a probability transform or selection rule. For next-token generation, the final logits usually have one score per vocabulary item for the current position.

M

Matrix
mathA two-dimensional tensor. Matrix multiplication is common in neural networks, but not every parameter or activation is stored or executed as a simple dense matrix.
Mixed-Precision Quantization
quantizationUsing more than one numeric encoding or precision across tensors or operations. In a model-file recipe, the mixture is an implementation decision that must be read from the pinned quantizer logic or artifact metadata—not inferred from a suffix alone.
Mixture of Experts MoE
architectureAn architecture in which routing selects a subset of expert sub-networks for each token. Total stored parameters, active parameters, memory traffic, and quantization choices must be distinguished.

O

Offset reconstruction offset, bias in some APIs
quantizationAn additive quantity that shifts a reconstruction grid. An implementation may store an offset directly or derive an equivalent shift from a scale and integer zero point; field names and units must be read from the exact scheme.
Output Layer LM head
architectureThe projection from a final hidden representation to vocabulary logits in many language models. It may share parameters with the input embedding or use a separate tensor.

P

Parameter
modelA learned value stored in a checkpoint, including weights and sometimes biases, scales, embeddings, or other learned tensors. Parameter count is not the same as artifact bytes or runtime memory.
Pass-to-Pass P2P
evaluationThe share or count of existing tests that passed before a change and still pass afterward. It is a regression-preservation measure and must be reported with its denominator and verifier revision.
Pareto Point
decisionA candidate for which no measured alternative is at least as good on every chosen objective and strictly better on one. The objectives and uncertainty—quality, latency, memory, energy, or others—must be stated.
Perplexity
evaluationThe exponential of average negative log-likelihood over a token sequence. Lower is better on the measured distribution, but a small perplexity change does not guarantee unchanged downstream behavior.

Q

Quantization
quantizationRepresenting values or computations with a restricted numeric encoding. Always identify the target—weights, activations, cache, or another tensor—plus mode, code width, grouping, calibration/importance procedure, and runtime support.
Quantized Target
quantizationThe values changed by quantization, such as weights, activations, KV cache, or selected tensors. “An 8-bit model” is incomplete unless the target and exceptions are stated.

R

Retained Runtime State
runtime stateState kept across generation steps so the engine does not recompute all prior work. A KV cache is one architecture-specific example; recurrent state and state-space state are different.
Reward
evaluationA scalar outcome defined by a particular evaluator. In strict software-engineering evaluation it may be binary and require every F2P and P2P condition to pass; never infer its definition from the word alone.
RMSNorm Root Mean Square Layer Normalization
architectureA normalization operation that rescales a feature vector using its root-mean-square magnitude, typically without subtracting the feature mean. Exact epsilon, learned scale, placement, and dtype are architecture and implementation details; RMSNorm is not interchangeable with LayerNorm.
Runtime Memory
measurementMemory allocated while the engine is loaded or generating: parameters, retained state, temporary workspaces, allocator overhead, executable resources, and sometimes duplicated buffers. Report the measurement boundary and whether it is peak or steady state.

S

Scalar
mathA single value, often described as a rank-zero tensor in tensor libraries.
Scale Factor
quantizationA value used to map between stored codes and reconstructed values. Schemes differ: some also use offsets/biases, zero points, codebooks, multiple scales, or nonuniform encodings.
Symmetric Quantization
quantizationA linear mapping whose reconstruction levels are centered around zero and normally use a scale without a separate shift. It can represent zero exactly but may use the available code range inefficiently for asymmetric value distributions.
Softmax
mathA function that maps finite scores to positive values that sum to one. It is used in conventional attention and often in converting logits to probabilities, though sampling can apply additional transformations.
Static Memory loaded parameter memory
measurementMemory attributable to loaded model tensors under a declared engine and loading policy. It differs from on-disk artifact size and excludes runtime growth unless explicitly included.

T

Tensor
mathA multidimensional array with a shape and data type. Rank zero is a scalar, rank one a vector, and rank two a matrix; layouts and sparse/packed representations add implementation detail.
Thinking Budget
generation settingA provider- or model-specific limit or control on reasoning-token use. It is not a universal Transformer field, and increasing it does not guarantee better results.
Thought Duplication
evaluationAn operational label for repeated reasoning content. It is meaningful only with a preregistered detection rule, unit of analysis, and validation; it must not be assumed to be caused by quantization without controlled evidence.
Token
modelAn integer ID produced by a tokenizer for a text or byte sequence. Token boundaries and vocabulary size depend on the exact tokenizer revision.
Transformer
architectureAn architecture introduced in 2017 that composes attention, position-wise transformations, residual paths, and normalization. Many LLMs use Transformer variants, but recurrent, state-space, linear-attention, and hybrid language models also exist.

V

Vector
mathA one-dimensional tensor. Its values have meaning only in the context of the operation, basis, and learned model.
Vocabulary vocab
modelThe set of token IDs defined by a tokenizer/model pair. The exact tokenizer files and revision are part of checkpoint parity in a controlled comparison.

W

Weight
modelA learned parameter used in an operation. Weight-only quantization changes selected stored weights while leaving activations or retained state at their configured formats unless stated otherwise.

Z

Zero Point
quantizationThe integer code corresponding to real zero in a common affine parameterization. It shifts code alignment; it is not the scale, and an API's stored floating bias/offset may be algebraically related without being the same quantity or unit.
