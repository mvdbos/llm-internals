Reference: LLM Glossary

Every term from the course, plus essentials. Print-friendly.

A B C D E F G H I K L M O P Q S T W

A

Attention aka Self-Attention
architecture The mechanism that lets each token "look at" every other token in the sequence to gather context. A token representing "bank" will attend strongly to "river" or "money" depending on surrounding words.
Attention is computed as: Query · Key → scores → softmax → weighted sum of Values. This is the core innovation of the Transformer architecture.
Attention Head
architecture One independent attention computation. Modern LLMs use multi-head attention — typically 32–64 heads running in parallel, each focusing on different relationships (syntax, semantics, long-range dependencies, etc.).

B

bf16 aka Brain Float 16
precision A 16-bit floating-point format (1 sign + 8 exponent + 7 fraction bits). Same dynamic range as FP32 but half the memory. The standard training/inference format for many modern LLMs. Often used interchangeably with FP16 for inference purposes.
Bias
architecture A learnable constant added to a layer's output, alongside the weight multiplication. output = weights × input + bias. Much smaller than weights in parameter count. Some modern architectures omit biases entirely.
Block-wise Quantization
quantization Quantizing weights in small groups (blocks), each with its own scale factor. Typical block size: 32 or 128 weights. A block of 32 weights at 4-bit uses: 32 × 4 bits = 128 bits + one 16-bit scale factor = 144 bits total (4.5 effective bits/weight).

C

Context Window
inference The maximum number of tokens the model can "see" at once. Determines how much conversation or code fits in a single prompt. Modern models: 32k–1M tokens. Longer context = more KV cache memory needed.

D

Deep-SWE
benchmark A benchmark for evaluating LLMs on real-world software engineering tasks (fixing GitHub issues). Uses pier + mini-swe-agent to run models against actual codebases with verifiable test suites.
d_model aka Hidden Dimension
architecture The size of the vector representing each token inside the model. Common values: 4096 (7B models), 8192 (70B+ models). Every tensor flowing through the transformer layers has this as its inner dimension.

E

Embedding
architecture A dense vector (list of numbers) that represents a token's meaning. The embedding layer converts token IDs (integers) into these vectors. "Cat" and "kitten" get similar embedding vectors; "cat" and "car" get different ones.
Embedding Layer
architecture The first layer of an LLM. A lookup table: token ID → dense vector of size d_model. Size = vocab_size × d_model. Highly sensitive to quantization because errors here affect which tokens the model can distinguish.

F

Feed-Forward Network (FFN)
architecture The "thinking" part of each transformer layer. A two-layer MLP that transforms each token's representation independently (no mixing between tokens — that's attention's job). Typical structure: d_model → 4×d_model → d_model.
FFN layers hold ~65% of all parameters in a transformer. They're the most redundant and therefore the most tolerant of aggressive quantization.
FP16 aka Half-Precision, float16
precision 16-bit floating-point: 1 sign + 5 exponent + 10 fraction bits. 2 bytes per weight. The practical quality baseline for LLM inference — virtually indistinguishable from FP32.
FP32 aka Single-Precision, float32
precision 32-bit floating-point: 4 bytes per weight. ~7 decimal digits of precision. Used during training, overkill for inference. A 7B model at FP32 = 28 GB.

G

GGUF aka GPT-Generated Unified Format
format The file format used by llama.cpp for distributing quantized models. A single file containing both the model architecture metadata and the quantized weights. Superseded GGML.
Group-wise Quantization
quantization See Block-wise Quantization. The terms are used interchangeably.

H

Hidden State
architecture The tensor flowing between layers, representing the current "understanding" of each token. Shape: (batch, sequence_length, d_model). Each transformer layer reads the hidden state, transforms it, and passes it forward.

I

Importance-Weighted Quantization (IQ)
quantization A smarter quantization strategy that gives more precision to "important" weights (those with larger magnitude or higher sensitivity) and fewer bits to less important ones. IQ quants (IQ4_XS, IQ3_XXS) typically outperform traditional Q quants at the same file size.
Inference
general Using a trained model to generate output (as opposed to training). Running model.generate("Hello") is inference. Inference is where quantization matters — training uses full precision.

K

K-quant
quantization A quantization scheme in llama.cpp that uses different block sizes and bit allocations per layer type. Attention layers get more bits; FFN layers get fewer. The "K" stands for the superblock structure used internally. Q4_K_M uses ~4.6 effective bits averaged across all layers.
KV Cache
inference A memory buffer that stores the Key and Value tensors from every previous token. Without it, the model would re-compute attention for the entire sequence on every new token (quadratic cost). The KV cache grows linearly with context length and is a major memory consumer alongside model weights.

L

Layer
architecture A mathematical operation that takes a tensor as input and produces a tensor as output. Layers contain learnable weights. Stacked together, they form the model.
Layer Norm
architecture A normalization layer that keeps values in a stable range (mean ≈ 0, std ≈ 1). Prevents values from exploding or vanishing as they pass through many layers. Has very few parameters — almost negligible for quantization purposes.
Logits
architecture The raw, unnormalized scores for each possible next token, output by the final layer. Shape: (batch, sequence, vocab_size). Converted to probabilities via softmax. The token with the highest logit is (usually) the model's next-word prediction.

M

Matrix
math A 2-dimensional tensor: rows × columns. Most layer weights are stored as matrices. A typical attention weight matrix in a 7B model: 4096 × 4096 numbers.
mini-swe-agent
benchmark The agent harness used in Deep-SWE benchmarks. It gives an LLM a bash shell and a coding task, then lets it edit files, run tests, and iterate. Tracks the trajectory (thoughts + actions) for analysis.
Mixed-Precision Quantization
quantization Using different bit-widths for different layers. K-quants implement this: embeddings at ~6-bit, attention at ~5-6 bit, FFN at ~4 bit. Maximizes quality per byte by putting precision where it matters most.
MoE aka Mixture of Experts
architecture A variant where FFN layers are split into multiple "experts," and each token only activates a subset (typically 2 out of 8). Reduces compute per token while increasing total parameters. DeepSeek-V2/V3, Mixtral, and recent Qwen models use MoE.

O

oMLX
inference An inference engine built on Apple's MLX framework for running LLMs on Apple Silicon. Supports on-the-fly quantization (switch between bf16 and quantized without re-downloading models). Used in the Deep-SWE benchmarks described in this course.
Output Layer aka LM Head
architecture The final layer: converts the last hidden state into logits over the vocabulary. Size = d_model × vocab_size. Often shares weights with the embedding layer ("tied embeddings"). Highly sensitive to quantization.

P

Parameter
architecture A single learnable number in the model. Includes weights and biases. "7B model" = 7 billion parameters. Each parameter is a weight value that gets multiplied during inference.
Perplexity
evaluation A measure of how "surprised" a model is by text. Lower = better. A perplexity of 10 means the model is as uncertain as if it were choosing randomly among 10 equally likely options at each step. Quantization increases perplexity slightly — Q8_0 adds ~0.01-0.02; Q4_K_M adds ~0.05-0.15.

Q

Q2_K, Q3_K_M, Q4_K_M, Q5_K_M, Q6_K, Q8_0
quantization Standard GGUF quantization levels. The number = average bits per weight. K = K-quant superblock structure. M/S/L = size variant (medium/small/large — more or fewer weights at higher precision). _0 = legacy fixed-block format. See Quantization Formats Reference for the full table.
Quantization
quantization The process of reducing the precision of model weights — e.g., from 16-bit floats to 4-bit integers. Saves memory (4× smaller) at the cost of slightly "fuzzier" weights. Modern techniques use per-block scaling to minimize quality loss.

S

Scalar
math A 0-dimensional tensor: a single number. Example: 3.14.
Scale Factor
quantization The high-precision (FP16) number stored alongside each block of quantized weights. It tells you how to map the stored integers back to real values: real_value ≈ scale × integer_value. One scale factor per block is what makes quantization work.
Softmax
math A function that converts a vector of arbitrary numbers into probabilities (all positive, sum to 1). Used in attention to compute "how much should I attend to each token" and at the output to get next-token probabilities.

T

Tensor
math A multi-dimensional grid of numbers. 0D = scalar, 1D = vector, 2D = matrix, 3D+ = higher-order tensor. In ML, "tensor" is used as the universal term for any multi-dimensional array of numbers flowing through the model.
Thinking Budget
inference The maximum number of tokens a model can use for its internal chain-of-thought before producing a final answer. Increasing it gives the model more room to reason — but only if the reasoning is coherent. A quantized model with a large thinking budget may just produce more fuzzy thoughts, not better ones.
Thought Duplication
evaluation When an LLM repeats the same reasoning pattern across multiple steps in an agentic loop. Measured as: unique_thoughts / total_thoughts. A ratio of 1.0 means clean, non-repeating reasoning. Values below 1.0 indicate the model is looping or re-treading old ground — a common failure mode for quantized models on long-horizon tasks.
Token
general The basic unit of text that an LLM processes. Not quite a word: "unbelievable" might be 3 tokens ("un", "believe", "able"). A model's vocabulary typically has 30k–250k tokens. Input text is tokenized before entering the embedding layer.
Transformer
architecture The neural network architecture underlying all modern LLMs. Introduced in 2017 ("Attention Is All You Need"). Core building block: alternating self-attention and feed-forward layers, wrapped with residual connections and layer normalization.

V

Vector
math A 1-dimensional tensor: a list of numbers. Example: a word embedding like [0.2, -0.5, 0.8, ...] with d_model elements.
Vocabulary aka Vocab
general The set of all tokens the model knows. Common sizes: 32k (Llama), 50k (GPT), 128k+ (modern multilingual models). The embedding table has one row per vocabulary entry.

W

Weight
architecture A learnable multiplier inside a layer. During inference: output = weight × input (+ bias). Weights are the vast majority of parameters in an LLM and are the primary target of quantization.
