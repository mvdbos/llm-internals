Suffix decoder ring


[Q|IQ][bits]_[K?][S|M|L|XS|XXS]

Q   = traditional linear quantization (evenly spaced buckets)
IQ  = importance-weighted (more precision for important weights)
K   = per-layer-type block sizes (K-quant superblock)
S   = small variant (fewer weights at higher precision)
M   = medium variant (the default sweet spot)
L   = large variant (more weights at higher precision)
XS  = extra-small (IQ only, more aggressive)
XXS = extra-extra-small (IQ only, most aggressive)
_0  = legacy format with fixed block size 32 (no K-quant)

Memory formula (approximate)


model_size_GB ≈ (num_params × bits_per_weight / 8) + overhead

Where:
  num_params is in billions
  bits_per_weight is the effective average (e.g., Q4_K_M ≈ 4.6)
  overhead ≈ 0.3-0.5 GB for small models, 0.5-2 GB for large models

Example: Qwen3.6-35B at Q4_K_M
  ≈ (35 × 4.6 / 8) + 1.0
  ≈ 20.1 + 1.0
  ≈ 21 GB
