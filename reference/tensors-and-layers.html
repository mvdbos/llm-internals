Key tensor shapes in an LLM

TensorTypical shapeWhat it is
Input tokens(batch_size, seq_len)Token IDs for each sequence in the batch
Embeddings(batch, seq, d_model)Each token converted to a dense vector (e.g. 4096 dims)
Attention Q/K/V(batch, heads, seq, d_head)Query, Key, Value projections per attention head
Hidden states(batch, seq, d_model)Output of each transformer layer
Logits(batch, seq, vocab_size)Raw scores for every possible next token

Layer types in a transformer

LayerWhat it doesHas weights?
EmbeddingToken ID → dense vector✅ Embedding table (vocab × d_model numbers)
Self-AttentionEach word "looks at" all other words to gather context✅ Q, K, V, and output projection matrices
Feed-Forward (FFN)Per-position non-linear transformation. The "thinking" part.✅ Two large weight matrices (d_model → 4×d_model → d_model)
Layer NormNormalizes values to stable ranges✅ Small: just scale + bias per dimension
Output (LM head)Final hidden state → vocabulary probabilities✅ Projection matrix (d_model × vocab_size)

Where the parameters live

In a typical 7B transformer model:

ComponentApprox. % of params
Feed-Forward layers (all of them)~65%
Attention layers (Q/K/V/O projections)~30%
Embedding + Output~5%
Layer Norms<0.1%

This distribution matters for quantization: the FFN layers hold most of the weights, so they dominate both memory usage and precision sensitivity.

Memory formula

Model memory ≈ num_parameters × bytes_per_parameter

FP32: 4 bytes/param → 7B model ≈ 28 GB
FP16: 2 bytes/param → 7B model ≈ 14 GB
Q8:   1 byte/param  → 7B model ≈  7 GB (+ small overhead)
Q4:   0.5 bytes/param → 7B model ≈ 3.5 GB (+ overhead)
Q2:   0.25 bytes/param → 7B model ≈ 1.75 GB (+ overhead)

Actual usage is higher: add ~1-2 GB for KV cache, activations, and framework overhead.
